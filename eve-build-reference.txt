EVE — BUILD REFERENCE
Last updated: Session 4
========================

WHAT EVE IS
-----------
Eve is a story organization and development workspace for writers.
No AI calls inside the app. No content generation. Eve is a coach, organizer, and craft educator.
Claude is used only to BUILD the app — not inside it.

TECH STACK
----------
Framework:   Next.js App Router (no src/ directory, no TypeScript)
Language:    JavaScript only
Styling:     CSS variables in app/globals.css + Tailwind
Fonts:       Playfair Display (headings) + Source Sans 3 (body)
Colors:      Forest green (#2D5016) + white (#FAFCF7)
Database:    Supabase — NOT YET WIRED. All data is mock/useState.
Hosting:     Vercel — NOT YET DEPLOYED.

CRITICAL CSS RULE
-----------------
Google Fonts @import MUST come before @import "tailwindcss" in globals.css.
Reversing this order causes a build error.

RULES (never break these)
--------------------------
- No emojis anywhere in UI
- No AI API calls inside the app
- Full file content every time — no ellipsis, no placeholders
- Never delete working code to simplify — only add
- npm run build must pass before git push
- All data is mock until Supabase is wired

FILE NAMING NOTE
-----------------
Next.js requires route files to be named page.js.
Component functions are named descriptively (e.g. ProjectsPage, IdeasPage).
Reference files use descriptive names (projects-page.js, learn-page.js) for download clarity.
When placing in VS Code, the actual file must be named page.js.
