SESSION 5 DIAGNOSIS — CRITICAL FINDING
======================================

PROBLEM: Supabase is completely absent from the codebase.
Confirmed by scanning all 15 loaded JS chunks on the live site —
zero chunks contain "supabase", "SUPABASE", or "createClient".
The Supabase client is never instantiated. Auth page is UI-only shell.

ROOT CAUSE (most likely): @supabase/supabase-js was never installed
OR lib/supabase.js exists but is never imported anywhere.

FIX — DO THESE IN ORDER:
--------------------------
1. In project root terminal:
   npm install @supabase/supabase-js

2. Create lib/supabase.js:
   import { createClient } from '@supabase/supabase-js'
   const supabaseUrl = process.env.NEXT_PUBLIC_SUPABASE_URL
   const supabaseAnonKey = process.env.NEXT_PUBLIC_SUPABASE_ANON_KEY
   export const supabase = createClient(supabaseUrl, supabaseAnonKey)

3. Create .env.local in project root:
   NEXT_PUBLIC_SUPABASE_URL=https://your-project-id.supabase.co
   NEXT_PUBLIC_SUPABASE_ANON_KEY=your-anon-key-here
   (Get both from: Supabase Dashboard > Project > Settings > API)

4. Add same env vars to Vercel:
   Vercel Dashboard > EVE project > Settings > Environment Variables
   Add for: Production + Preview + Development

5. Wire auth page first — add to app/auth/page.js:
   import { supabase } from '@/lib/supabase'
