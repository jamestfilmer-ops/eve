SECTION 6: ENGINEERING PRINCIPLES
======================================

BEFORE WRITING ANY CODE
------------------------
1. Run: git status — confirm clean working tree
2. Fetch the current file from GitHub raw URL before editing
3. Identify exactly which lines change — nothing else
4. Ask: will this break any other file that imports this?

MAKING CHANGES
--------------
5. One logical change per commit
6. Always provide full file content — no ellipsis, no shortcuts
7. npm run build locally before git push — never push broken code
8. Never rename/move/delete files without updating all imports
9. Add 'use client' to any page using event handlers or useState

COMMITTING
----------
10. Always commit package-lock.json alongside package.json for new installs
11. After staging: verify git status shows the right files
12. If git says "nothing to commit" — those files were already committed,
    check their actual content instead of trying to re-add

DEBUGGING HIERARCHY
--------------------
1. git status — are changes actually committed?
2. Vercel build logs — did deploy succeed?
3. Fetch raw GitHub file — is the code what you think it is?
4. Browser JS chunks — is the code in the bundle?
5. Console errors on live site
6. Supabase dashboard — are env vars correct, are rows appearing?

SUPABASE RULES
--------------
- NEXT_PUBLIC_ prefix required for all client-side env vars
- .env.local must NOT be committed
- Vercel env vars set separately from .env.local
- After new env vars in Vercel: must trigger a new deployment
- package-lock.json must be committed for Vercel to install new packages
- emailRedirectTo must be set on signUp or confirmation links go to localhost
- Two accounts exist — always verify URL matches the right project

REGRESSION PREVENTION
----------------------
- Read the file before editing — never write from memory
- After globals.css change: check fonts still load
- After Nav.js change: check all 5 nav links work
- After auth change: test sign in AND sign up
- Never clean up mock data until Supabase wiring is confirmed working
