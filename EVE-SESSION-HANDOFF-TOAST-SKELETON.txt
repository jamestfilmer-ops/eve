BUILD FIX HISTORY (for future reference)
════════════════════════════════════════════════════════════

Root cause of all build failures this session:
  Next.js App Router prerenders pages at build time — even 'use client'
  pages. Any hook (useState, useContext, etc.) that runs at module
  evaluation time on the server causes a "Invalid hook call" crash.

Three separate fixes required:
  1. Toast.js: dismiss() defined after addToast() → addToast called
     undefined dismiss → crash at module eval. Fixed by flipping order.
  2. layout.js: importing ToastProvider directly into a server component
     crashes /_not-found. Fixed with Providers.js wrapper.
  3. Pages: dashboard, ideas, auth all needed export const dynamic =
     'force-dynamic' to prevent SSR prerender attempts.

Engineering rule going forward:
  ANY page that uses useToast(), useRouter(), or useSearchParams()
  needs export const dynamic = 'force-dynamic' at the top.
